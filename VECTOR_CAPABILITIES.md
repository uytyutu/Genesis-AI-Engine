# Vector — живой паспорт способностей

**Источник правды** для команды, тестировщиков и CEO.  
Обновлять **после каждого merge**, который даёт пользователю новую реальную способность.

**Правила:**
1. **Одна способность = законченная ценность** (не «создал файл» / «записал текст» отдельно).
2. **Дорожная карта = сценарии**, не модули архитектуры.
3. **Один merge — одна новая способность** (или завершение шага сценария с видимым результатом).

## Сценарии (roadmap)

### Сценарий 1 — Создание сайта

**Пользователь пишет:** «Создай сайт стоматологии.»

**Ожидаемый результат (когда READY):**
- ✓ Workspace создан
- ✓ brief.md
- ✓ index.html, style.css, assets/
- ✓ preview (ссылка)

| Шаг | Статус |
|-----|--------|
| Документ / brief в workspace | 🔄 (часть «создать документ» ✅) |
| Структура сайта (HTML/CSS) | ⬜ |
| Factory / preview | ⬜ |
| **Сценарий целиком** | 🔄 IN PROGRESS |

---

### Сценарий 2 — Анализ документов

**Пользователь пишет:** «Вот бизнес-план» (+ PDF).

**Ожидаемый результат (когда READY):**
- ✓ PDF открыт и прочитан
- ✓ структура + SWOT + риски
- ✓ report.md в workspace

| Шаг | Статус |
|-----|--------|
| PDF intake в промпт (базовый) | ✅ (Brain, не Execution) |
| Полный анализ → report.md | ⬜ |
| **Сценарий целиком** | ⬜ PLANNED |

---

### Сценарий 3 — Разработка / исправление ошибки

**Пользователь пишет:** «Исправь ошибку.»

**Ожидаемый результат (когда READY):**
- ✓ логи → файл → правка → тесты → diff

| Шаг | Статус |
|-----|--------|
| Чтение файлов workspace | 🏗 executor есть, фраза в чате — ⬜ |
| Git / terminal / тесты | ⬜ |
| **Сценарий целиком** | ⬜ PLANNED |

## Примеры для тестировщиков

### Создай README

| Поле | Значение |
|------|----------|
| **Статус** | ✅ READY |
| **Фраза** | `Создай README` |
| **Ожидание** | Ответ с `✓ Готово`, `README.md`, `provider: execution` |
| **Проверка API** | `POST /api/public/genesis-ai` |

### Создай сайт стоматологии

| Поле | Значение |
|------|----------|
| **Статус** | 🔄 IN PROGRESS |
| **Фраза** | `Создай сайт стоматологии` |
| **Сейчас** | Текстовый ответ Brain / partial plan — **не** полный сайт |
| **Следующий merge** | Commit 2: структура сайта в workspace |

### Проанализируй бизнес-план

| Поле | Значение |
|------|----------|
| **Статус** | ⬜ PLANNED |
| **Фраза** | PDF + «проанализируй бизнес-план» |
| **Сейчас** | Не обещать SWOT / report.md |

### Привет, как дела?

| Поле | Значение |
|------|----------|
| **Статус** | ✅ READY (диалог) |
| **Ожидание** | Обычный ответ Brain / Groq, не execution |
