# Vector — живой паспорт способностей

**Источник правды** для команды, тестировщиков и CEO.  
Обновлять **после каждого merge**, который даёт пользователю новую реальную способность.

## Правила развития

1. **Одна способность = законченная ценность** — не «создал файл» / «записал текст» отдельно.
2. **Дорожная карта = сценарии**, не модули архитектуры.
3. **Один merge — одна новая способность** (или завершение шага сценария с видимым результатом).
4. **Capability must compose** — каждая способность возвращает стандартный объект (`workspace_id`, `artifact_id`, `files`, `artifacts`, `preview_url`, `logs`, `status`), чтобы другие агенты могли строить цепочки без переписывания.
5. **Product Truth** — способность **существует** только если её может проверить человек через публичный `/site`, без терминала, Owner API и знаний архитектуры. Нужен Mission Control или логи → это ещё инфраструктура, не продукт.
6. **Human Gate** — не начинать Commit N+1, пока Commit N не проверен **реальным человеком на beta** (`/site`, одна фраза, глазами). Зелёные unit-тесты ≠ READY.

| Способность | Статус | Проверка на `/site` |
|-------------|--------|---------------------|
| **Создавать документ по запросу** | ✅ READY | `Создай README` → `✓ Готово`, файл в workspace |

## Три первых сценария (типы работы)

### 1. Development — «создавать»

**Фраза-тест:** `Создай сайт стоматологии`

**Ожидаемый результат (READY):**
```
✓ Анализирую запрос...
✓ Создаю Workspace...
✓ Формирую структуру проекта...
✓ Создаю brief...
✓ Генерирую HTML...
✓ Генерирую CSS...
✓ Создаю preview...
✓ Готово.
```
**Артефакты:** `brief.md`, `index.html`, `style.css`, `assets/`, `preview/` + кнопка preview.

| Шаг | Статус |
|-----|--------|
| Workspace + brief + HTML/CSS + assets | ✅ |
| Preview в браузере из чата | ✅ |
| **Сценарий Development** | 🔄 HUMAN GATE (код в ветке, beta после deploy) |

---

### 2. Documents — «анализировать»

**Фраза-тест:** `Проанализируй бизнес-план` (+ PDF)

**Ожидаемый результат (когда READY):**
- SWOT, риски, финансы, рекомендации → `report.md`

| Шаг | Статус |
|-----|--------|
| PDF intake в промпт (базовый) | ✅ (Brain) |
| Полный анализ → report.md | ⬜ |
| **Сценарий Documents** | ⬜ PLANNED |

---

### 3. Engineering — «изменять»

**Фраза-тест:** `Исправь ошибку`

**Ожидаемый результат (когда READY):**
- логи → файл → правка → тесты → diff

| Шаг | Статус |
|-----|--------|
| Чтение файлов workspace | 🏗 executor есть |
| Git / terminal / тесты | ⬜ |
| **Сценарий Engineering** | ⬜ PLANNED |

## Примеры для тестировщиков (только `/site`)

### Создай README

| Поле | Значение |
|------|----------|
| **Статус** | ✅ READY |
| **Фраза** | `Создай README` |
| **Ожидание** | `✓ Готово`, `README.md`, `provider: execution` |

### Создай сайт стоматологии

| Поле | Значение |
|------|----------|
| **Статус** | 🔄 HUMAN GATE (после deploy на beta) |
| **Фраза** | `Создай сайт стоматологии` |
| **Ожидание** | прогресс ✓, список файлов, кнопка **Открыть preview** |
| **Preview** | реальный HTML-сайт (не текст в чате) |

### Проанализируй бизнес-план

| Поле | Значение |
|------|----------|
| **Статус** | ⬜ PLANNED |
| **Сейчас** | не обещать SWOT / report.md |

### Привет, как дела?

| Поле | Значение |
|------|----------|
| **Статус** | ✅ READY (диалог) |
| **Ожидание** | Brain / Groq, не execution |

## Composable output (`generate_site`)

```json
{
  "workspace_id": "ws-…",
  "artifact_id": "site-…",
  "files": ["brief.md", "index.html", "style.css", "assets/", "preview/"],
  "artifacts": [{ "id": "…", "kind": "bundle", "path": "artifacts/preview" }],
  "preview_url": "/api/public/execution/preview/{workspace_id}",
  "logs": ["Анализирую запрос", "…"],
  "status": "completed"
}
```

Цепочка будущего: анализ бизнес-плана → `generate_site` → публикация → презентация — без переписывания `generate_site`.
