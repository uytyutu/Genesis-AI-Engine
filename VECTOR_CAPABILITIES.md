# Vector — живой паспорт способностей

**Источник правды** для команды, тестировщиков и CEO.  
Обновлять **после каждого merge**, который даёт пользователю новую реальную способность.

## Правила развития

1. **Одна способность = законченная ценность** — не «создал файл» / «записал текст» отдельно.
2. **Пирамида ценности** — roadmap по уровням полезности, не случайные фичи и не модули архитектуры.
3. **Один merge — одна новая способность** (или завершение шага с видимым результатом).
4. **Capability must compose** — каждая способность возвращает стандартный объект (`workspace_id`, `artifact_id`, `files`, `artifacts`, `preview_url`, `logs`, `status`), чтобы другие агенты могли строить цепочки без переписывания.
5. **Product Truth** — способность **существует** только если её может проверить человек через публичный `/site`, без терминала, Owner API и знаний архитектуры.
6. **Human Gate** — не начинать Commit N+1, пока Commit N не проверен **реальным человеком на beta** (`/site`, одна фраза, глазами). Зелёные unit-тесты ≠ READY.

## KPI (с Commit 3)

> **Сколько часов реальной человеческой работы Virtus Core может заменить?**

| Способность | Оценка экономии |
|-------------|-----------------|
| Создать README | ~5–15 мин |
| Создать лендинг (сайт) | ~30–120 мин |
| Анализ бизнес-плана | ~2–8 ч *(цель Commit 3)* |
| Исправление типовой ошибки | ~1–4 ч *(цель Commit 4)* |

Не «сколько возможностей», а **сколько работы снято с человека**.
