# Virtus Core — Product Architecture v1.0

**Статус:** LOCKED (архитектура продукта) · **не код** · **не замена** `ROADMAP.md` / `WHY.md`  
**Верхний закон:** `Docs/Virtus_Core_Product_Constitution_v1.md` — при конфликте побеждает Constitution.  
**Дата:** 2026-07-13  
**Миссия сейчас:** Mission 1 — первый € (только Уровень 1 · Гость)

> **Девиз платформы:** мы оптимизируем не количество функций, а **количество сомнений до покупки**.

---

## 1. Что строим

**Virtus Core** — не набор AI-функций. Это **экосистема продукта** с одним движком принятия решений:

```text
Customer Decision Engine (B-001 LOCKED)

Цель → Выбор → Вопрос → Preview Δ → … → Покупка → Производство
```

Один механизм. Меняются только вопросы и ветки (сайт, бот, Stripe, автоматизация, исправление…).

**Vector** — мир клиента (простой путь к результату).  
**Mission Control** — мир владельца (деньги, проекты, Guardian, AI Employees). Клиент **никогда** не видит MC.

---

## 2. Два мира

| | **Vector** (клиент) | **Mission Control** (владелец) |
|---|---|---|
| **Задача** | Привести к покупке и развитию бизнеса | Управлять компанией Virtus Core |
| **Язык** | Результат, не технология | Метрики, риски, доход |
| **Примеры** | «Получить клиентов», preview сайта | Stripe, прибыль, Guardian, заказы |
| **Правило** | Не показывать финансы, Guardian, аналитику | Не смешивать с клиентским UI |

---

## 3. Customer Decision Engine (LOCKED)

### Главное правило

```text
Vector никогда не спрашивает: «Что вы хотите написать?»
Vector спрашивает: «Какой следующий выбор приближает к результату?»
```

### Правила экрана

| Правило | Критерий провала |
|---------|------------------|
| **3 секунды** | «Что здесь вообще происходит?» |
| **Одно действие** | Больше одного главного CTA на экране |
| **Preview Δ** | После ответа ничего не изменилось визуально |
| **Покупка** | Нельзя купить, когда «уже похоже на мой продукт» |
| **Экран** | Нельзя ответить: «Почему это приближает к покупке?» → экран удалить |

### Свободный текст

Только где необходимо: название, телефон, email, адрес. Остальное — кнопки, карточки, переключатели, 📎 загрузка.

### Blind ×2 (обязателен для закрытия Slice)

1. **Предприниматель:** непонятно? закрыл бы? что удержало?  
2. **Инженер:** один разрыв → один фикс → снова весь путь.  
3. **Вопрос:** «Стало ли легче купить?» — нет → Slice открыт.

`Engineering ✅` только с доказательством. Иначе `⏳`.

## 4. Роли пользователей

```text
Гость → Клиент → Подписчик (уровень бизнеса) → Владелец (CEO / Mission Control)
```

| Роль | Кто | Где живёт | Mission 1 |
|------|-----|-----------|-----------|
| **Гость** | Первый визит, ещё не платил | `/site` · Vector Guided Commerce | **ДА — фокус** |
| **Клиент** | Купил продукт (сайт, бот…) | «Мои проекты», статус производства | После первого € |
| **Подписчик** | Платит за уровень бизнеса | Vector post-purchase | Horizon |
| **Владелец** | Ramish / команда | Mission Control | Параллельно, не в клиентском UI |

## 5. Уровни экосистемы

### Уровень 1 — Гость (Mission 1 · СЕЙЧАС)

```text
Заходит
  ↓
Vector: цель (карточки, не чат)
  ↓
Guided Conversation: один вопрос · один экран
  ↓
Preview живёт и меняется после каждого шага
  ↓
«Оплатить» (когда доволен — не обязательно весь путь)
  ↓
Stripe → производство → заказ в Mission Control
```

**Definition of Success (Mission 1):** человек без опыта, без свободного текста (кроме необходимого), за 5–10 минут видит свой продукт, ценность и кнопку «Оплатить».

**Реализация:** только B-001 Slices по LOCKED-контракту. Не весь мастер сразу.

---

### Уровень 2 — Клиент (после первого €)

После покупки гость **не** видит «Купить сайт». Видит:

```text
Здравствуйте, {имя}.
Ваш сайт готов на 82%.
Сегодня рекомендую:
  ✓ подключить Google
  ✓ добавить AI
  ✓ закончить контакты
```

**Мои проекты**

```text
Website · Статус · Производство
```

**Upsell — не навязывание, следующий логичный шаг:**

```text
Хотите усилить сайт?
○ AI-консультант
○ Telegram
○ SEO
○ Google Business
○ Онлайн-запись
```

Пример ритма:

- Неделя 1: «Появились посетители — добавить AI-консультанта?»
- Месяц 2: «Принимать оплату? → Stripe»
- Далее: Telegram → CRM → AI Employee…

> Продаём не подписку. Продаём **следующий уровень бизнеса**.

**Gate:** первый реальный клиент + первый €.

---

### Уровень 3 — Подписка как уровень бизнеса (Horizon)

Не «тариф с фичами», а **уровень зрелости бизнеса клиента**:

| Уровень | Смысл для клиента | Ориентир возможностей |
|---------|-------------------|------------------------|
| **Free** | Попробовать | 1 сайт, без AI / Telegram / Automation |
| **Starter** | Работающий бизнес онлайн | 2 проекта, AI, Stripe, обновления |
| **Business** | Рост и автоматизация | AI Employees, Guardian, Automation, Telegram, CRM |
| **Enterprise** | Несколько компаний / филиалов | API, White Label, мульти-сотрудники |

**Gate:** подтверждённый product-market fit, не Mission 1.

## 6. Универсальный каталог продуктов

Один **Commerce Engine**, разные ветки вопросов:

| Продукт | Цель клиента (не технология) | Пример первого вопроса |
|---------|------------------------------|-------------------------|
| 🌐 Сайт | Получать клиентов / продавать онлайн | Название компании |
| 🤖 AI-консультант | Отвечать клиентам 24/7 | Для чего нужен помощник? |
| 💬 Telegram-бот | Продажи / поддержка в мессенджере | Для чего бот? |
| ⚙️ Автоматизация | Экономить время | Что занимает больше всего времени? |
| 💳 Платежи | Принимать оплату | Уже есть Stripe? |
| 🔧 Исправление | Починить то, что сломалось | Что не работает? |
| 📈 SEO / Google | Найти клиентов в поиске | Где вы сейчас в Google? |

Алгоритм ветки всегда:

```text
Цель → вопросы (минимум) → preview Δ → результат → цена → Оплатить → производство
```

## 7. Локализация бизнеса (не перевод)

Vector определяет **страну** (IP / выбор / домен) и **молча** готовит юридическую оболочку:

**Германия**

```text
✓ Impressum
✓ Datenschutzerklärung
✓ Cookie Banner
✓ DDG / DSGVO
```

**Польша / США / …** — свои наборы. Клиент не обязан знать термины; Vector объясняет человеческим языком («страница, которую требует закон в вашей стране»).

**Связь с Mission 1:** Legal / L-001 уже в карте. В Guided Flow — отдельный Slice после Blind PASS базового пути.

---

## 8. Логика рекомендаций (Vector)

```text
Событие (трафик, ошибка, возраст проекта, пробел в данных)
  ↓
Vector: один следующий логичный шаг (не список из 10)
  ↓
Клиент принимает или откладывает
  ↓
Preview / проект обновляется
```

**Guardian** анализирует паттерны (много спрашивали SEO → предложить услугу) — **рекомендует**, не публикует. Владелец одобряет в Mission Control.

---

## 9. Mission Control (владелец)

Клиент не видит. Примеры панели «Сегодня»:

```text
Доход сегодня · 143 €
Stripe · 7 новых клиентов
3 сайта · 2 Telegram · 1 Automation
Guardian: SSL истекает · AI нашёл проблему
```

После оплаты гостя MC получает: заказ, клиента, доход, статус производства.

---

## 10. Что в Mission 1 vs Horizon

| В scope Mission 1 | Horizon (после первого €) |
|-------------------|---------------------------|
| B-001 Guest path → первая покупка | Уровень 2 «Мои проекты» |
| Customer Decision Engine по Slice | Upsell-ритм, подписки |
| Preview + Stripe + производство (срезами) | Guardian-рекомендации услуг |
| Legal / payee / Impressum (L-001) | Полная локализация по странам |
| Blind ×2 на каждый Slice | Mission Control как центр роста |

**Vision Freeze:** не добавлять отделы и модули без слота и gate. Этот документ — **карта**, не разрешение строить всё.

---

## 11. Текущее состояние B-001 (честно)

```text
Customer Decision Engine    LOCKED (этот документ + контракт)
Slice 1 (цель, не чат)        Engineering ⏳ · Blind ⏳
Slice 2 (preview, логотип)    Engineering ⏳ · Blind ⏳
Slice 3+                      не открыт до Blind PASS
```

Следующий Slice — только из Blind Diary владельца или Blind ×2 с одним разрывом.

---

## 12. Главный критерий платформы

Не: *«Сможет ли он разобраться?»*  
А: **«Сколько минут до желания нажать „Оплатить“?»**

Vector не продаёт сайт. Vector помогает человеку **принять решение о покупке** — уменьшая сомнения на каждом шаге.
