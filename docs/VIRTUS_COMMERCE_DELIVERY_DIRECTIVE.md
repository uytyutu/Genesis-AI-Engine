# Core Principle

Customers do **not** buy AI.

Customers buy completed outcomes.

Virtus Core exists to remove technical complexity from creating, publishing and continuously improving digital products.

**Sell capabilities, not subscriptions as the primary frame.** The customer thinks *"I need a website for my business"* — not *"I need the Pro tier."*

---

# Two Different Products

Virtus Core offers two complementary services.

## Product

A project that belongs to the customer.

Examples:

* website
* mobile application
* desktop software
* AI system
* automation
* game
* future digital products

After payment the customer owns everything included in the purchased deliverable.

Typical deliverables include:

* source code
* project files
* documentation
* generated assets
* deployment package
* configuration
* installation instructions

**Ownership never expires.**

> **The customer never loses ownership because the subscription expires.**

---

## Service

Everything required to keep the project alive.

Examples:

* publishing
* hosting
* deployment
* SSL
* backups
* monitoring
* updates
* SEO improvements
* analytics
* continuous development
* future feature requests

These services may be included in subscriptions.

**Subscriptions never revoke ownership.**

Subscriptions are **not** for unlocking files.

Subscriptions are for **ongoing collaboration with Vector**.

# Commerce Flow

The desired customer journey is:

```
Idea
↓
Conversation
↓
Co-Creation
↓
Live Draft
↓
Preview
↓
Approval
↓
Price
↓
Payment
↓
Production Build
↓
Delivery
↓
Publishing
↓
Continuous Collaboration
```

The customer should never pay before understanding what is being created.

### Trust-based payment

Not:

```
Pay now.
```

Instead:

```
We have built a concept together.
↓
Are you happy with this direction?
↓
Would you like me to build the complete production version?
↓
Price
↓
Payment
```

Vector sells through trust — never through pressure.

# Publishing Philosophy

A completed project should not leave customers wondering:

> "What do I do now?"

Instead Vector should continue guiding them.

Example:

> "The project is ready. Would you like to publish it now?"

Generation is **not** the end of the relationship. It is the beginning.

---

# Publishing Options

## If the customer has no domain

Offer:

* register a new domain
* connect later
* temporarily publish under a Virtus Core subdomain

Example:

```
company.virtuscore.app
```

This allows immediate sharing.

## If the customer already owns a domain

Vector helps connect it.

Possible approaches:

* automatic integration where supported
* guided DNS configuration
* step-by-step instructions

The customer should never feel abandoned after delivery.

---

# Subscription Philosophy

Subscriptions are not for unlocking files.

Subscriptions are for ongoing collaboration.

Examples:

* improve existing website
* redesign pages
* add online shop
* improve SEO
* translate into new languages
* build new modules
* monitor uptime
* manage deployments
* continuous optimization

A lapsed subscription may pause collaboration — it must never revoke ownership of what was already purchased.

### Capability tiers (hypothesis — not final pricing)

Subscriptions may reflect **depth of collaboration**, not message limits:

| Level | Direction (example) |
|-------|---------------------|
| **Free** | Meet Vector |
| **Creator** | Create projects together |
| **Pro** | Multiple specialists, stronger models, priority |
| **Business** | Digital team, shared workspace, clients |

Concrete limits are defined **after** first real users — not before.
