# Virtus Core — Service Contracts

**Статус:** CEO утверждено 2026-07-10  
**Единица выпуска:** **Услуга (Service Release)** — не commit, не capability  
**Порядок выпуска:**

```text
RC1 (Gates 3–5 → WC=1)
  ↓
SR-18  Workspace OS
  ↓
SR-1   Business Plan Pro
  ↓
SR-2   Website Pro
  ↓
SR-5   Presentation Pro
```

**North star услуг:** не 100 услуг — **10 лучших в мире**. Сейчас в работе: 4.

**Virtus Core = Digital Company.** Клиент «нанимает» команду специалистов; одно лицо — **Vector**.

**Язык команды (с SR-18):** на совещаниях и в документах говорим только:
- «Выпускаем услугу **Workspace OS**»
- «Выпускаем услугу **Business Plan Pro**»
- «Выпускаем услугу **Website Pro**»
- «Выпускаем услугу **Presentation Pro**»

# Rule №10 — WOW Gate (все услуги)

Перед статусом READY CEO или тестировщик без контекста отвечает:

> **«Вау» или «ну, прикольно»?**

«Прикольно» = **не выпускать**. Доработать до WOW из контракта ниже.

| Услуга | WOW (ожидаемая реакция) |
|--------|-------------------------|
| **Workspace OS** | «О, вот мои файлы, цепочка работы — это не чат» |
| **Business Plan Pro** | «Он не только проанализировал — переписал, v2 PDF/DOCX, список правок, готовность» |
| **Website Pro** | «Preview уже открыт, mobile, ZIP, можно править — не просто HTML» |
| **Presentation Pro** | «Готовая презентация из моего плана — не bullets в чате» |

WOW проверяется **на beta**, не pytest.

# SR-18 — Workspace OS

## Название
**Workspace OS** — рабочее пространство Virtus Core

## Что обещаем пользователю
Вы видите **всю свою работу** в одном месте: документы, отчёты, сайты, история. Vector — помощник справа, не главный экран.

## Что получает пользователь
- ✔ экран «Последняя работа» с цепочкой (PDF → Summary → Report → Site)
- ✔ список артефактов в workspace (открыть, скачать)
- ✔ история сессий / проектов
- ✔ preview сайта и отчётов **внутри** Virtus Core
- ✔ чат Vector как панель, не как «пустой чат»

## Что НЕ обещаем
- ✔ облачную синхронизацию между устройствами (аккаунты)
- ✔ совместное редактирование командой
- ✔ замену Google Drive / Notion

## PASS
Пользователь открывает `/site`, **сразу видит последнюю работу и файлы**, может открыть отчёт и сайт без поиска в чате.

## FAIL
Пользователь видит только чат и «Пока пусто», хотя файлы уже созданы в workspace.

## Human Gate (beta)
1. Выполнить анализ PDF → появились артефакты в UI workspace  
2. Создать сайт → сайт виден рядом с отчётом  
3. Закрыть вкладку → открыть снова → та же «Последняя работа»  
4. Чат доступен, но не занимает 80% экрана на desktop

# SR-1 — Business Plan Pro

## Название
**Business Plan Pro** — профессиональная доработка бизнес-плана

## Что обещаем пользователю
Мы **исправляем** ваш бизнес-план и отдаём **новый документ**, не советы в чате.

## Что получает пользователь
- ✔ новый **PDF** (Business Plan v2)
- ✔ новый **DOCX** (Business Plan v2)
- ✔ Executive Summary
- ✔ бизнес-заключение (HTML)
- ✔ список изменений (что исправлено)
- ✔ оценку готовности (0–100)
- ✔ приоритетные рекомендации
- ✔ следующие шаги (intent actions)

## Что НЕ обещаем
- ✔ регистрацию фирмы
- ✔ открытие банковского счёта
- ✔ получение финансирования или гарантию одобрения инвестором
- ✔ юридическую силу документа
- ✔ замену аудитора / налогового консультанта

## PASS
Пользователь загружает PDF → получает **скачиваемые v2 PDF и DOCX** + список изменений. Executive Summary читается за 30 секунд и отвечает: «запускать или дорабатывать».

## FAIL
Пользователь получает только текст в чате или Markdown без нового файла.

## Human Gate (beta)
1. Реальный PDF бизнес-плана (RU / DE / EN)  
2. Услуга «Business Plan Pro» завершена — есть v2 PDF + DOCX  
3. Список изменений соответствует diff (не выдуманные правки)  
4. Язык v2 = язык исходного документа  
5. Нет смешения RU/EN/DE в одном отчёте

# SR-2 — Website Pro

## Название
**Website Pro** — создание и доработка сайта из workspace

## Что обещаем пользователю
Сайт создаётся **из вашего проекта** в workspace, лежит рядом с бизнес-планом, его можно редактировать, скачать и посмотреть.

## Что получает пользователь
- ✔ сайт (HTML/CSS) в workspace
- ✔ preview внутри Virtus Core
- ✔ reuse данных из Business Plan (если был анализ)
- ✔ редактирование через Vector («измени заголовок»)
- ✔ ZIP экспорт
- ✔ мобильно адаптивная вёрстка
- ✔ базовые SEO meta

## Что НЕ обещаем (до отдельного SR)
- ✔ публикацию на ваш домен (live hosting) — отдельный этап после EL3
- ✔ уникальные фото / бренд-дизайн уровня агентства
- ✔ интернет-магазин, личный кабинет, оплату на сайте
- ✔ гарантию позиций в Google

## PASS
Цепочка в workspace: **Бизнес-план → (исправлен) → Создать сайт → сайт рядом → Preview → ZIP**. Explain Reuse показывает, что взято из анализа.

## FAIL
Сайт не связан с проектом, пользователь заново вводит нишу и услуги, workspace пуст.

## Human Gate (beta)
1. После Business Plan / анализа в том же workspace: только «Создай сайт»  
2. Сайт отражает данные плана  
3. Preview работает  
4. ZIP скачивается  
5. Редактирование одной секции через чат меняет файл

# SR-5 — Presentation Pro

## Название
**Presentation Pro** — презентация для инвестора из вашего проекта

## Что обещаем пользователю
Из бизнес-плана и отчёта в workspace — **готовая презентация** (файл), не outline в чате.

## Что получает пользователь
- ✔ deck (PDF и/или PPTX)
- ✔ структура: проблема, рынок, продукт, финансы, команда, ask
- ✔ данные из Executive Summary и report
- ✔ лежит в workspace рядом с планом и сайтом

## Что НЕ обещаем
- ✔ гарантию pitch success
- ✔ анимации уровня Keynote designer
- ✔ автоматический pitch инвесторам

## PASS
Пользователь с готовым отчётом в workspace: «Создай презентацию» → скачиваемый файл → слайды соответствуют цифрам из плана.

## FAIL
Только bullet list в чате без файла.

## Human Gate (beta)
1. Workspace уже содержит report / summary  
2. Запрос презентации → файл в artifacts  
3. Минимум 8 слайдов, цифры из исходного документа
