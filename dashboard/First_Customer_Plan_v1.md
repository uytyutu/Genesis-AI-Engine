## Framing: не продажа сайта — первый партнёр Genesis

Mission 1 ищет **первого партнёра**, а не «покупателя лендинга».

| Старый фрейм | Фрейм Genesis |
|--------------|---------------|
| «Мы делаем сайты» | «Мы заметили, чем можем помочь — покажем пример, потом обсудим сотрудничество» |
| Транзакция | Доверие → результат → долгосрочные отношения |
| Клиент | Партнёр |

**Preview в Mission 1:** вручную (скрин/макет главной страницы, 15–30 мин на бизнес). **Preview Engine — Horizon**, не сейчас.

## Objective

Получить **первого реального платящего партнёра** (незнакомец, реальные €, не тест `4242`) и подтвердить рыночную ценность Genesis.

**Evidence Level:** переход **EL2 → EL3** (MVP работает → первый платящий партнёр). Следующий модуль — только после EL3.

**KPI Mission 1 (из Public Launch):**

> Незнакомый человек открыл ссылку на телефоне, оформил заказ и смог оплатить услугу.

## 1. Канал привлечения (один)

### Выбранный канал: **персональный outreach к локальным МСБ, которым можно быстро принести пользу**

**Формат:** поиск по признакам слабого онлайн-присутствия → персональное сообщение (Email / WhatsApp) → **показать пример** → при интересе → `/order`.

**География старта:** **небольшие города** (меньше конкуренции) — Pirna, Bautzen, Freital, Görlitz, Radebeul. Dresden — после первого €. **Berlin — не старт.**

**Сегмент:** стоматология, кафе, автосервис, салон красоты, репетитор, малый бизнес 1–5 человек.

**Источники (не только Google Maps):** Maps, Instagram, Gelbe Seiten, локальные каталоги, витрины без сайта.

### Критерии отбора — «кому можно быстро помочь»

В список попадает бизнес с **≥2 признаками** (отмечать в таблице):

| Признак | Пример |
|---------|--------|
| Устаревший сайт | дизайн 2010-х, Flash, битые ссылки |
| Нет мобильной версии | нечитаемо с телефона |
| Нет HTTPS | `http://` в адресе |
| Нет онлайн-записи | только телефон в часы работы |
| Нет WhatsApp / кнопки связи | только email на общий ящик |
| Нет формы заявки | нельзя оставить запрос вне звонка |
| Только Facebook / Instagram | нет собственного сайта |

**Не писать всем подряд** — только тем, где видна конкретная, быстрая польза.

## 2. Почему именно этот канал

| Критерий | Оценка |
|----------|--------|
| Бюджет | €0 на рекламу |
| Скорость | 7–14 дней до первого ответа |
| Соответствие продукту | Landing €350–650 = типичная боль «нет нормального сайта» |
| Измеримость | Контакты → **ответы** → диалог → `/order` → € |
| «Незнакомец» | Cold outreach = не друзья и не тестовые клики |
| Purpose | Сокращает путь от возможности до устойчивого результата |
| Доверие | Партнёрский подход + пример до оплаты |

**Почему не другие каналы сейчас:**

| Канал | Почему не первый |
|-------|------------------|
| Google Ads | Нужен бюджет + время на обучение; нет baseline конверсии |
| SEO / контент | Месяцы до трафика |
| Product Hunt / маркетплейсы | Не целевая аудитория для локального лендинга |
| Соцсети (органика) | Медленно; нет аудитории Genesis |
| Тёплая сеть | Быстрее, но не доказывает «незнакомец» |

## 4. Текст сообщений (два шага)

### Шаг 1 — первый контакт (без ссылки на оплату)

#### DE

**Betreff:** Kurze Idee für [Geschäftsname] — [Stadt]

```
Guten Tag [Name],

ich habe [Geschäftsname] in [Stadt] gefunden und bemerkt, dass [konkretes Signal: z.B. die Seite auf dem Handy schwer lesbar ist / kein HTTPS / nur eine Facebook-Seite].

Ich habe einen kurzen Entwurf vorbereitet, wie Ihre Online-Präsenz moderner wirken könnte —
ohne Verpflichtung, nur als Beispiel.

Darf ich Ihnen den Entwurf schicken? Wenn er Ihnen gefällt, besprechen wir gerne die Zusammenarbeit.

Freundliche Grüße
[Dein Name]
Genesis
[Kontakt-E-Mail]
```

#### RU

```
Здравствуйте, [Имя]!

Нашёл [название] в [городе]. Заметил: [конкретный признак — мобильная версия, HTTPS, форма заявки и т.д.].

Подготовил короткий пример, как это могло бы выглядеть — без обязательств, просто чтобы показать идею.

Могу прислать? Если понравится — обсудим сотрудничество.

[Имя]
Genesis
[email]
```

### Шаг 2 — после «да» / интереса

1. Отправить **ручной preview** (PNG/PDF или 2–3 скрина).
2. Коротко: что улучшится для их клиентов (запись, звонок, доверие).
3. Если готовы: ссылка на заказ с фиксированной ценой:

```
Paket ab 350 €, Online-Zahlung, Lieferstatus:
https://genesis-ai-engine.vercel.app/order
```

### Правила сообщения

- **1 персональная строка + конкретный признак** — обязательно.
- **Не обещать** то, что Factory ещё не делает.
- **Не скидки** в первом эксперименте.
- **Первое сообщение без `/order`** — оплата после доверия.
- **Без массового спама** — только отобранные бизнесы с реальной пользой.

## 5. Live Stripe Checklist

Выполнить **до** первого outreach незнакомцу.

### Stripe Dashboard

- [ ] Аккаунт Stripe **активирован** (KYC / бизнес-данные)
- [ ] Переключиться в **Live mode** (не Test)
- [ ] Скопировать **Live Secret Key** `sk_live_...`
- [ ] Создать **Live Webhook**:
  - URL: `https://genesis-ai-engine-production.up.railway.app/api/webhooks/stripe`
  - Event: `checkout.session.completed`
  - Скопировать **Live Signing Secret** `whsec_...`

### Railway (backend)

- [ ] Заменить `STRIPE_SECRET_KEY` на `sk_live_...` (не test key)
- [ ] Заменить `STRIPE_WEBHOOK_SECRET` на live `whsec_...`
- [ ] Проверить: `RESEND_API_KEY`, `GENESIS_EMAIL_FROM` на месте
- [ ] Дождаться redeploy (~2 мин)
- [ ] Проверить: `GET /api/sales/payment-status` → `configured: true`

### Юридическое / доверие (минимум EU)

- [ ] `contact_email` в `public_launch.json` на production volume
- [ ] Клиент видит контакт на сайте или в письме
- [ ] Понимание: вы оказываете **услугу** (сайт), не SaaS-подписку — для чека/Impressum

### Smoke test (после live)

- [ ] Один заказ **своей картой** на минимальный пакет €350 (можно refund через Stripe Dashboard)
- [ ] Подтвердить: статус «В работе», email от Genesis, Stripe Dashboard → Payment succeeded

**Только после этого** — outreach незнакомцам.

## 6. KPI Mission — First Customer Experiment

### Объём эксперимента

| Метрика | Цель |
|---------|------|
| Контактов отправлено | **25** |
| **Ответов** | **≥5** |
| **Созвонов / содержательной переписки** | **≥2** |
| **Оплат (live €)** | **≥1** |
| Срок | **14 дней** |
| Каналов | **1** (outreach МСБ, партнёрский) |

> **Ответы важнее оплат на этом этапе.** 0 ответов из 25 = проблема сообщения или сегмента, не продукта.

### Воронка (гипотеза)

| Этап | Консервативно | Цель |
|------|---------------|------|
| Отправлено | 25 | 25 |
| **Ответов** | 2–3 | **≥5** |
| **Диалог / созвон** | 1 | **≥2** |
| Preview отправлен | 1–2 | ≥3 |
| Переход на `/order` | 1 | ≥2 |
| **Оплата (live €)** | 1 | **≥1** |

### Критерий успеха Mission 1

```text
✅ ≥1 незнакомый партнёр
✅ реальный платёж (live Stripe)
✅ письмо от Genesis получено
✅ Factory запущен / статус «В работе»
```

### Диагностика по этапу (где теряется интерес)

| Симптом | Вероятная причина | Действие |
|---------|-------------------|----------|
| 0 ответов / 25 | Сообщение, сегмент, город | Сменить текст или нишу → v1.2 |
| Ответы есть, 0 диалогов | Preview слабый или не убедительный | Улучшить ручной preview |
| Диалог есть, 0 `/order` | Цена / доверие / не та боль | Уточнить на созвоне |
| Заказ есть, 0 оплаты | Техника / Stripe / трение | Диагностика воронки |
| ≥1 оплата | — | **Mission 1 Complete** → PSS → GOS v1 |

### Когда считать эксперимент неудачным

**После 14 дней И 25 контактов:**

| Исход | Решение |
|-------|---------|
| **0 ответов** | Сменить сегмент, город или текст — **Mission 1 продолжается** |
| **<5 ответов** | Итерация сообщения, не продукт |
| Ответы есть, 0 оплат | Диагностика по таблице выше |
| ≥1 оплата | Ретроспектива + **Partner Success Score** |

**Не расширять scope** — менять **гипотезу**, не архитектуру.

## 7. Данные после первого клиента (Learning Engine / ретроспектива)

Собрать в течение 48 часов после оплаты (таблица или заметки):

### О клиенте и пути

1. **Откуда пришёл?** (канал, город, как нашли)
2. **Почему купил?** (цена, скорость, доверие, боль)
3. **Что едва не остановило?** (цена, сомнения, язык, оплата)
4. **Какие вопросы задавал?** (до и после оплаты)
5. **Сколько стоило привлечение?** (время в часах × вашу ставку; € на рекламу = 0)
6. **Сколько времени от первого контакта до оплаты?** (дни)
7. **Сколько времени до старта Factory / доставки?** (часы)
8. **Доволен ли клиент?** (1–10, цитата)
9. **Купил бы снова / порекомендует?** (да/нет/почему)
10. **Что улучшить в следующем эксперименте?** (1–3 пункта)

### Partner Success Score (PSS) — после доставки

Внутренний показатель **0–100** (не прибыль, а устойчивость отношений):

| Компонент | Вес | Как измерить |
|-----------|-----|--------------|
| Удовлетворённость | 30% | Оценка 1–10 после сдачи |
| Цели достигнуты | 25% | Да/частично/нет — что хотел партнёр |
| Порекомендует | 25% | Да/нет + готовность дать отзыв |
| Купит снова | 20% | Да/нет в горизонте 12 мес |

**Формула (упрощённо):** PSS = средневзвешенная нормализованная оценка.

Один довольный партнёр с высоким PSS ценнее разовой продажи — фиксировать для Learning Engine (обезличенно).

### О системе Genesis (внутренняя ретроспектива миссии)

- Правильно ли была выбрана цель?
- Не сделали ли сложнее, чем нужно?
- Можно ли было быстрее?
- Чему научилась система?
- Как это изменит Mission 2?

### Запреты на данных

- Не использовать идеи клиента A для клиента B (Принцип доверия).
- В глобальную память — только **обезличенная статистика** (например: «конверсия outreach стоматология 4%»).
